## 医疗健康信息平台基础——集成体系 ![ 信息集成体系 ](img/xxjctx.png)   ## 互联互通技术架构 + 标准统一使用 + 数据统一采集 + 接口统一制定 + 应用统一整合 + 门户统一集成   ## 标准体系 遵循国际标准、国家标准、行业标准、地方标准、自定义标准规范。 `①` 主数据（标准代码、基础数据） → `②` 数据元 → `③` 数据集 → `④` 业务模型toAPI → `⑤` 服务授权 → `⑥` 共享服务 ![ 标准体系 ](img/xxbztx.png)   ## 数据融合支撑平台 ![ 数据融合 ](img/xxztjg.png) 信息资源的整合与共享服务，促进信息标准化，实现信息互联互通。 数据融合建设主要涉及数据采集与数据交换、数据加工融合处理，以及数据共享服务应用三个主要环节，每个环节中都有相应的平台系统支撑。 　　以下为数据融合各支撑系统架构，主要由前置中间件、数据融合平台、服务集成平台、云平台管理门户及其相关组件构成。 `前置中间件`：解决线上线下异构系统的数据通讯问题。帮助用户快速建立IT应用系统间的数据交换，支撑用户数据集成应用的快速实现，提供可按需流程定制及内外网之间的数据采集与交换业务、物联数据实时交换、可远程配置及监控管理等。 `数据融合平台`：数据中心可以对任意多个线下或线上系统或数据源进行数据采集。 + 对采集至数据中心的业务数据，根据数据之间的业务关联性进行比对、清洗、转换、加工等数据处理，以提高业务数据质量； + 以管理对象唯一标识为主索引，关联融合其相关业务信息，建立管理对象融合信息库； + 采用数据集模型、业务模型驱动并基于元数据管理、主数据管理，对数据及信息资源进行规范化、标准化管理； + 对上层应用的复杂分析计算需求，进行任务化的配置管理，由任务执行引擎自动完成。 + 数据融合平台提供多个模块以实现上述功能，包括元数据模块，数据集模块，业务模型模块，主数据模块，在线编码模块等等。 `服务集成平台`：将所有服务规范后，统一接入，形成统一的服务目录对外提供服务，同时标准化服务调用通讯接口和协议，提供在线API IDE工具，通过支持API服务的开发、多版本的API服务管理、API服务的发布、API服务的上线审核、团队开发的API服务，帮助现有的开发团队和产品平滑地迁移到PaaS平台环境。建立按需服务，做到可管、可监、可控、可用，进而对服务进一步组织、整合，生成新的、满足更多更深层次需求的服务，提高信息资源的使用效率。 `云平台管理门户`：实现对中间件、服务集群、数据服务及授权认证等进行集中配置、监控和管理，提高自动化运维水平，让运维人员能够随时掌握云平台的运行状态并能够及时发现和捕获异常。   ## 特点 `跨平台`：支持常见的Linux、Windows及Unix操作系统，可部署运行在物理/虚拟Linux、Windows服务器、普通PC机、前置机等设备之上。 `数据库`：支持MySQL、MS SQLServer、Oracle、DB2。 `集群部署`：支持动态横向集群扩展，提供Linux及Windows集成安装包，安装后进行简单配置即可运行。 `热部署`：节点启动后可通过负载管理自动加入运算队列。部署调试过程中无需停机，各节点及节点内的各服务运行互不干扰。 `易用性`：简单易用学习成本低。实施人员经过简单培训即可快速上手应用。 `高性能、高可用`：底层基于互联网技术，支持大并发应用场景，采用国际最先进的动态集群运算技术（无需部署业务应用软件），通过动态增加服务器数量提高运算和并发能力。
