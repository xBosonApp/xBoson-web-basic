xBoson-MS

微服务开发平台

  • 　　提供在线云端集成开发环境，在线代码级别开发，实现快速响应需求的变化，随时可进行业务调整，无需重新编译、停机、部署调试等过程，基本实现所见即所得开发效率。
  • 产品功能特点
    • 跨平台：运行环境支持常见的 Linux、Windows 及 Unix 操作系统，可部署运行在物理/虚拟 Linux、Windows 服务器。
    • 数据库：支持 Oracle、SQL Server、MySQL、MongoDB、H3C-MPP。
    • 集群部署：后端计算引擎无业务应用耦合，支持快速动态横向集群扩展。
    • 模型驱动：将业务逻辑抽象出来形成业务模型数据服务供其他组件调用，从而实现快速开发同时提高业务模型复用性减少重复开发。
    • 视图模板引擎：视图模板服务是由平台提供服务集群，解决动态 UI 视图开发需要。它将 UI 视图模板化后与业务模型服务层及数据服务层的业务逻辑组件进行动态包装或重组后，以多种视图服务的形式对外暴露 UI 视图，提供给第三方系统或者客户端调用。
    • 开发工具：提供在线云端代码级别开发，无需安装浏览器插件，有浏览器的设备即可随时开发服务，实现快速响应需求的变化，随时可进行业务调整，无需重新编译、停机、部署、调试等过程，基本实现所见即所得的开发效率。
    • 易用性：采用 JavaScript 编程，学习成本低，开发人员经过简单学习即可快速上手开发接口服务。
    • 高性能、高可用：采用轻量级微内核，支持大并发应用场景，采用动态集群运算服务（无需部署业务应用软件），通过动态增加服务器数量提高运算和并发能力。
    • 项目管理：具备完整的项目管理功能，实现不同项目的代码隔离和人员隔离，以及API服务和 UI 的开发、测试、上下线等功能，开发过程中的代码与已上线代码互不影响。

xBoson-PaaS

基础支撑PaaS平台

  • 　　大数据集成项目建设主要涉及数据采集与数据交换、数据加工融合处理，以及数据共享服务应用三个主要环节，每个环节中都有相应的平台系统支撑。
  • 　　以下为数据融合各支撑系统架构，主要由前置中间件、数据融合平台、服务集成平台、云平台管理门户及其相关组件构成。
  • 产品架构
    • 前置中间件：解决线上线下异构系统的数据通讯问题。帮助用户快速建立应用系统间的数据交换，支撑用户数据集成应用的快速实现，提供可按需流程定制及内外网之间的数据采集与交换业务、物联数据实时交换、可远程配置及监控管理等。
    • 数据融合平台：数据中心可以对任意多个线下或线上系统或数据源进行数据采集。对采集至数据中心的业务数据，根据数据之间的业务关联性进行比对、清洗、转换、加工等数据处理，以提高业务数据质量；以管理对象唯一标识为主索引，关联融合其相关业务信息，建立管理对象融合信息库；采用数据集模型、业务模型驱动并基于元数据管理、主数据管理，对数据及信息资源进行规范化、标准化管理；对上层应用的复杂分析计算需求，进行任务化的配置管理，由任务执行引擎自动完成。数据融合平台提供多个模块以实现上述功能，包括元数据模块，数据集模块，业务模型模块，主数据模块，在线编码模块等等。
    • 服务集成平台：将所有服务规范后，统一接入，形成统一的服务目录对外提供服务，同时标准化服务调用通讯接口和协议，提供在线API IDE服务开发工具工具，支持多版本的API服务管理、API服务的发布、API服务的上线审核、团队开发的API服务，帮助现有的开发团队和产品平滑地迁移到PaaS平台环境。建立按需服务，做到可管、可监、可控、可用，进而对服务进一步组织、整合，生成新的、满足更多更深层次需求的服务，提高信息资源的使用效率。
    • 云平台管理门户：实现对中间件、服务集群、数据服务及授权认证等进行集中配置、监控和管理，提高自动化运维水平，让运维人员能够随时掌握云平台的运行状态并能够及时发现和捕获异常。
  • 产品特点
    • 跨平台：支持常见的Linux、Windows及Unix操作系统，可部署运行在物理/虚拟Linux、Windows服务器、普通PC机、前置机等设备之上。
    • 数据库：支持 Oracle、SQL Server、MySQL、MongoDB、H3C-MPP。
    • 集群部署：支持动态横向集群扩展及热部署，安装后进行简单配置即可运行，节点启动后可通过负载管理自动加入运算队列。
    • 易用性：简单易用学习成本低。实施人员经过简单培训即可快速上手应用。
    • 高性能、高可用：采用轻量级微内核，支持大并发应用场景，采用动态集群运算服务（无需部署业务应用软件），通过动态增加服务器数量提高运算和并发能力。
